Videos

We present a total of 2322 (Train: 1597, Val: 199, Test: 526) scene-text videos. The videos contain a combination of three different languages, namely, English, Hindi, and Marathi.

For each such scene-text, we create 12 videos using 12 different types of camera transformations, broadly categorized into 5 groups:

  1. four types of translation, that could be left, right, up and down,
  2. two types of roll, including clockwise and anti-clockwise,
  3. two types of tilt which could be up-down or down-up motion,
  4. two types of pan, that is left-right and right-left, and
  5. two types of zoom which could be in or out.

We use a camera with a tripod stand to record all these videos to have a uniform control. It is important to note that there are four types of translations, whereas there are only two types for all other transformations. We capture all these videos at 25 fps with a resolution of 1920 Ã— 1080. Table 1 shows the distribution of videos on the basis of transformations.

Annotation

train_ann.txt and val_ann.txt are tab seperated files. Each line contains the following data:

  • Video name
  • Mannually annotated as well as verified text annotation
  • Type of transformation - TR, R, T, P, Z (namely Translation, Roll, Tilt, Pan, Zoom)
  • Time where transformation starts. (Default: start of video)
  • Time where transformation ends. (Default: end of video)

S - Only 5 videos in Train annotations. \N in some annotations.

We also provide word-level mask for all video frames which we generate using Cloud Vision API. This is provided if needed for weak supervision. train_bbox and val_bbox contain a folder for each video. Each video folder contains JSON files for each frame of the video with word-level mask as given using Vision API. The first object in the JSON file is the mask for all the text in the frame. The following objects are mask for each individual word. We recommend using word-level masks as per the text annotation in train_ann.txt and val_ann.txt file. Note that the text annotations (by Cloud Vision API) and mask are not verified by us.
